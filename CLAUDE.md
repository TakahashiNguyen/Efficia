## 📋 Overview

**Efficia** is a lightweight, offline-first web editor for Microsoft Word / Excel / PowerPoint applications built with **Next.js 13+**, **React**, and **Tailwind CSS**. It enables real-time collaborative editing without any external file server or cloud storage - all data lives locally in the browser.

### Key Features
- **Offline-first architecture**: No files ever leave your machine; all data is stored locally on disk (in `metadata.json`)
- **Real-time sync via WebSocket/ZeroMQ**: Multiple clients can edit the same document simultaneously with instant updates
- **Customizable UI**: Uses ShadCN UI components generated by Tailwind CSS
- **Simple API**: Minimal JSON payloads for load, edit, and upload operations

## 🛠️ Getting Started

### 1️⃣ Install Dependencies

```bash
pnpm install tailwindcss postcss autoprefixer shadcn-ui   # UI components via ShadCN
pnpm add express socket.io                                 # Bridge server & client networking
```

**Note**: The project uses `shadcn-ui` for Tailwind-styled components, which are generated by the Next.js app. You can customize these components by editing files in `src/components/ui/`.

### 2️⃣ Run Development Server (hot reload)

```bash
pnpm dev
# Equivalent shortcut in pnpmrc
```

The server serves at **http://localhost:3000**, where you can open multiple browser tabs to edit different files simultaneously. All edits are instantly visible because the bridge uses WebSocket over a tiny JSON API (`api/file`). No external storage is involved - all data lives locally on disk in `metadata.json`.

### 3️⃣ Build for Production

```bash
pnpm build && pnpm serve
# Builds the Next.js app and serves it at http://localhost:3000 or wherever you deploy.
```

This step minifies assets, optimizes bundles, and produces a production-ready deployment ready to be hosted (e.g., Vercel, Netlify). No external data is written beyond this local process.

## 📂 Key Files & Their Purpose

| File | Purpose |
|------|---------|
| `src/app/page.tsx` | Entry point for each document type - renders the compiled bundle and offers "Edit" buttons |
| `src/api/file.ts` | Express route that serves `/public/<fileId>` instantly; also adds an upload endpoint (`POST /upload`) – note: data is never persisted beyond this request |
| `components/ui/*` (generated by ShadCN) | Tailwind-styled cards, buttons and text inputs you can customize further |
| `metadata.json`   *(optional)* | Small JSON file storing only a metadata record for each document; see §5 below |

## 🛠️ Architecture Overview

### Data Flow

1. **Load Document**: User clicks "Load" → AJAX request to `api/file` → Server forwards compiled Next.js bundle back to client
2. **Edit Text**: Click inside editor → React state updates → WebSocket message (`action: edit`) sent from `src/utils` to `/api/file` → Server broadcasts changes instantly
3. **Real-time Sync**: Every edit is broadcast to all other sockets listening for that file ID

### Communication Protocol

- **Client → Server**: JSON payloads (load, edit, upload requests)
- **Server → Client**: WebSocket messages (`action: edit`) with compiled bundle data
- **No external storage**: All data lives locally in `metadata.json` on the filesystem

## 🚀 Deploying

```bash
pnpm build   # builds the Next.js app + static assets (public folder)
cd public    # optional – you can move the generated files elsewhere if desired

# Run locally again with pnpm serve or deploy to Vercel, Netlify, etc.
```

*The server does **not** write any document content to disk.*  
If later you need persistence beyond this local editor (e.g., version history), simply add a backend service that reads/writes the same `metadata.json` file – no changes required on your side.
