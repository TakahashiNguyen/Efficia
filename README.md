# Efficia  
**A lightweight, offline-first web editor for Microsoft Word / Excel / PowerPoint**

---

## ⚡ Overview

Efficia lets you create, edit and share fully editable documents (Word/Excel/PPT) **without ever touching a remote file server or storing any user data on the cloud.**
All editing happens entirely in your browser using Next.js 13+ with Tailwind CSS + ShadCN UI components.

* No files are uploaded to any external service.*  
* Editing is performed locally; changes appear instantly for every connected client (real-time sync).

## 🔧 Getting Started

```bash
# 1️⃣ Create & enter project folder (uses pnpm)
pnpm create next-app@latest efficia --with-ts
cd efficia

# 2️⃣ Install core dependencies
pnpm i tailwindcss postcss autoprefixer shadcn-ui   # UI components via ShadCN
pnpm add express socket.io                         # bridge server & client networking 
```

### 📂 Project Skeleton

The generated scaffold already contains the following files:

| File | Purpose |
|------|---------|
| `src/app/page.tsx` | Entry point for each document (Word, Excel, PowerPoint). Shows a **pre‑rendered** bundle and offers "Edit" buttons. |
| `src/api/file.ts`  | Express route that serves `/public/<fileId>` instantly; also adds an upload endpoint (`POST /upload`) – note the data is never persisted beyond this request. |
| `components/ui/*` (generated by ShadCN) | Tailwind‑styled cards, buttons and text inputs you can customize further. |
| `metadata.json`   *(optional)* | Small JSON file storing only a metadata record for each document; see §5 below. |

## 🖥️ Editing Documents  

### 1. Load a Document

* From the UI click *"Load"* → an AJAX request (via Socket.io/ZeroMQ) is sent to `api/file`.  
* The server forwards that request back with the compiled Next.js bundle for `<fileId>`, which your page renders immediately.

```tsx
// src/app/page.tsx – simplified version

async function loadDocument() {
  await connect().send({ type: "load", file_id: fileId });
}
```

### 2. Edit Text / Cells  

* Click inside the editor → you can edit directly (React state updates).  
* Any change triggers a WebSocket message (`action: edit`) from `src/utils` to `/api/file`.  
* The server stores that event locally, then broadcasts it back as soon as possible.

> **No save button** is needed – edits are saved automatically when the user navigates away or closes the tab (the page component re‑renders).

### 3. Real‑time Collaboration  

If you enable WebSocket support:

```ts
// src/utils/connector.ts
import { io } from "socket.io-client";
export async function connect() {
  return await io("ws://localhost:4000/ws/:fileId");
}
```

* Every edit is broadcast to **all other sockets listening for that file**.  
* No server‑side state persists – every client has the full picture of who's editing which part.

### 4. Upload / Download (optional)

```bash
pnpm add next-mock-server   # or any static‑file router you like
```

You can replace `/api/file` with a simple Express route that serves:

* `GET /files/:id` → returns the compiled bundle for `<fileId>`.  
* `POST /upload` – receives binary data from drag‑and‑drop, saves it to your local disk (no external storage), and sets its ID.

## 📚 Key Features

| Feature | Implementation |
|---------|----------------|
| **Offline‑first** | All data lives in memory; never reaches remote servers. |
| **Real‑time sync** | WebSocket / ZeroMQ broadcast of edit events, with zero server persistence. |
| **File sharing** | One file ↔ many clients – each client receives every change instantly. |
| **Customizable UI** | ShadCN Tailwind components give a modern look; you can extend or replace them entirely. |
| **Simple API** | Minimal JSON payloads for load, edit, and upload (no auth). |

## 🛠️ Extending the Project

* **Add more document types** – just change `src/app/page.tsx` to accept different MIME‑type parameters (`file_type: "doc", "xlsx", …`).  
* **Integrate a real editor library** – replace Next.js with an external markdown/word processor (e.g., Typora, WPS Online) and still keep the same architecture. |
| **Implement version control** – add `git` integration to track changes locally; metadata.json can be backed by Git instead of this file if you ever need history beyond "loaded vs edited". |
* **Add authentication / permission checks** – on a production server, require an API token that grants read/write rights for specific files. |
